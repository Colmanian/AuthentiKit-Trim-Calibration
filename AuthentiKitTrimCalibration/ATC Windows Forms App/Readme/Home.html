AuthentiKit Tuning App

Experimental Release

This software maps game controllerphysical buttonsto virtual outputs that can be tuned for an optimal in-sim experience. Outputs can be virtual axes, or vitual buttons. A typical use would be mapping your AuthentiKit Sptifire Mk IX elevator trim wheel to a single virutal axis that you use in sim. What's more, you can tune the sensitivity of the axis to aproximate the correct turn rate of the trim wheel in sim.

Getting Started

1.  Install .NET 5.0 Runtime 

The Tuning App is written in a Microsoft software framework called .NET. You need to download and install the x86 runtime for desktop apps to use the AuthentiKit Tuning App.

Download .NET 5.0 Runtime

2.  Install vJoy

vJoy is free, open source softare that the Tuning App uses for its virtual outputs. You need to download and install specificly version 2.2.1, which you can find at the link below, or search online. Unfortunatly the installer is a little buggy and can hang at the end. If this happens, the installation was still probably sucessful - you just have to close the process in task manager. In the future, I'd like to remove our dependance on vJoy, but that might be more work than I can reasonably do in my free time. If you can help, see the links above.

Download vJoy 2.2.1

3.  Configure vJoy

  • Open the Configure vJoy app, which you can find this from the windows start menu, and typing "vJoy".
  • Ensure Enable vJoy is ticked
  • On Tab 1, Ensure that Axes X, Y and Z are checked as a minimum.
  • On Tab 1, Ensure that at at least 10 buttons are available/
  • Hit Apply if needed

Note: The vJoy software sometimes takes a few seconds to respond to clicks. If this happens, it probably hasn't crashed, just give it a minute.

4.  Configure Tuning App

Each virtual output is driven my a Mapping. There are two types of mapping.

  • Axis Mapping:  One input button moves the virutal axis up, and a different button moves the axis down. The sensitivity, is (for now) an arbetrary value that you set as a number.
  • Button Mapping: One input button is used to pulse a virtual output button several times, as specificed by the multiplier value.

You can Save and Load your configurations from the Config menu. The Presets menu provides default configurations for AuthentiKit setups using the Leo Bodnar BU0836A. The software looks for devices called AuthentKit or BU0836 Interface as inputs when loading a preset. You can modify any setting after loading a preset. So far, the presets are as follows.

  • FlyingIron Spitfire Mk. IX for MSFS

After you've set the configuration you want, you must click the Activate button for each mapping before you'll see the vJoy outputs move.

Note: If you're using the BU0836 Rotary Encoders Configuration tool as part of your AuthentiKit setup, the recommended pulse width is 32ms.

5.  Configure Simulator

The Virtual Output Axes/Buttons should be assigned to the relevant actions in your sim of choice. For example, for the FlyingIron Spitfire Mk. IX preset, you should map vJoy 1: Axis X to the Elevator Trim axis in MSFS.

Due to the nature of the pulses from the trim wheel encoder, it is not realistic to get an extact 1:1 mapping. You can absolutely get something that most people will find more than good enough for full imersion in VR though. 

Note: For sims that allow multiple inputs bound to the same sim actions (e.g. MSFS), be sure to unasign the input buttons from the sim, and only use the virtual axis that's being driven by the Tuning App.
