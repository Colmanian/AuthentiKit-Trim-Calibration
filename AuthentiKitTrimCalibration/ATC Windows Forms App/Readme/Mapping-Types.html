Mapping Types

Remap to Axis

Maps two joystick buttons to a Virtual Output Axis. One button moves the axis up (Input Button +) and the other moves the axis down (Input Button -) 

Sensitivity is an arbetrary measure that you should experiment with to find something that works for you. We've found that around 350 works well for mapping an Authentikit Spitfire Mk IX Elevator Trim Wheel to the FlyingIron Spitfire Mk IX in MSFS. 

Button Multiplier

Maps a single Input Button on a real peripheral, to a single Virtual Output Button in vJoy.
The behaviour of the output button can be tuned for optimal experience in sim.

  • Multiplier can be used to pulse the output button on and off several times each time the intput button is pressed.
  • Hold Threshold Start can be used to hold the output button in a steady on state. This is useful for sims like MSFS, where you need to hold an button down (rather than pulse it) in order to get maximum turn rate. If set to 500ms, the output will hold on if two pulses are received within half a second of eachother (500ms). 
  • Hold Threshold Stop is how long the the output is held on, after it goes into a hold state. If set to 1000ms, the output button will turn off after it hasn't received any input for a second (1000ms).

Remap to Axis (Advanced)

This is an experimental feature that takes raw encoder  input (rather than button presess) and maps that across to an axis. It can lead to better accuracy in the sim, but is a bit more complex to explain and set up. 

The typical use case would be an encoder connected to a Leo Bodnar BU0836A controller, such as an Authentikit Trim Wheel. Usually, you would use the Leo Bodnar BU0836_encoders.exe software to make the encoder appear as button presses in windows. Unlike the basic 'Remap to Axis' mapping, this one requires you to forgo the use of BU0836_encoders.exe, and provide the encoder pattern straight into the AuthentiKit Tuning app, as Encoder Input A,  and Encoder Input B. 

  • Encoder Pulses per Revolution can be found on the data sheet of your encoder. Values between 6 and 24 are supported here.
  • Revolutions In Per Revolution Out is how many turns you have to put in to span the full scale of the virtual output axis. A real spitfire's elevator trim wheel turns 4 times to go from full nose up to full nose down trim, hence this should be set to 4 for best mapping to the FlyingIron Spitfire Mk. IX in MSFS.
